# 챕터7에 대해서 정리해보자

1.추상메소드

추상 메소드는 선언되어 있으나 구현되어 있지 않은 메소드이다. abstract 키워드를 이용해 선언할 수 있다.

함수를 중괄호로 {} 정의하지 않고, 선언만 해놓은 것이다.

      public abstract void setName();
      public abstract void printName();
추상 메소드는 서브 클래스에서 오버라이딩하여 구현해야 한다.

(추상 메소드 = 설계만 해놓은 메소드, 구현은 서브 클래스에서 해야한다.)

2. 추상클래스

- 추상 메소드를 하나라도 가진 클래스를 의미한다. (추상 메소드가 없어도 abstract를 class앞에 붙여줘야 함)

- 추상 메소드가 없지만, abstract로 선언된 class를 의미한다.

- 하위클래스에서는 반드시 미완성된 메소드(abstract메소드)를 오버라이딩하여 완성해야 한다.
  (하위클래스에서 abstract메소드를 완성하지 않으면 하위클래스도 abstract클래스가 됨)
  
- 추상 클래스는 객체를 생성할 수 없다.


- 추상 클래스의 단순 상속

추상 클래스를 상속받아 추상 메소드를 구현하지 않으면 추상 클래스가 된다. 따라서 서브 클래스도 abstract로 선언해야한다.

- 추상 클래스 구현 상속

서브 클래스에서 슈퍼 클래스의 추상 메소드를 구현하는 방식이다. 오버라이딩을 통해 추상 메소드를 구현해 주면 추상 클래스가 아닌 일반 클래스로 사용할 수 있다.

4. 인터페이스
- 완벽한 abstract클래스를 interface라고 한다
  완벽한 abstract클래스 : 모든 메소드가 abstract메소드인 클래스를 의미한다.
  
  모든 메소드는 무조건 자동으로 public abstract로 선언된다.
- 모든 변수는 무조건 자동으로 public static final로 선언된다.
  
  public : 어디서든 접근가능/ static : 인스턴스생성없이 클래스명으로 접근가능 / final: 상수
- interface는 상속과는 다르게 다중 구현(implements)이 가능하면 extends와 같이 사용해도 된다.
  interface를 이용하면 다중상속의 효과를 낼 수 있다.


class Aa { ... }

interface Bb { ... }

interface Cc { ... }

interface Dd { ... }
class Test extends Aa implements Bb, Cc, Dd, { ... }

- 클래스디자인시에 사용됨
- interface를 implements한 클래스에서는 interface의 모든 메소드를 오버라이딩 해야 함
  오버라이딩 안하면 abstract클래스로 지정해야 함
  오버라이딩시 접근제어지시자는 무조건 public으로 지정해야 함


- interface의 용법
1. 클래스디자인시 사용(동시에 작업이 진행될 수 있게 함)
2. 다중상속의 일부지원(실제로는 사용할 일이 거의 없음)
3. public static final로 선언되는 interface의 변수를 사용

5.추상클래스와 인터페이스 비교

- 유사점

객체를 생성할 수 없고, 상속을 위한 슈퍼 클래스로만 사용된다. 클래스의 다형성을 실현하기 위한 목적으로 사용된다.

차이점


추상 클래스는 서브 클래스에서 필요로 하는 대부분의 기능을 구현하여 두고 서브 클래스가 상속받아 활용할 수 있도록 하되, 서브 클래스에서 구현할 수 밖에 없는 기능만을 추상 메소드로 선언하여, 서브 클래스에서 구현하도록 하는 목적을 가지고 있다. (다형성)

- 추상 메소드와 일반 메소드 모두 포함

- 상수, 변수 필드 모두 포함

인터페이스

인터페이스는 객체의 기능을 모두 공개한 표준화 문서같은 것으로, 개발자에게 인터페이스를 상속받는 클래스의 목적에 따라 인터페이스의 모든 추상 메소드드를 만들도록 하는 목적을 갖기고 있다.(다형성)

- 변수 필드는 포함하지 않음

- 상수, 추상 메소드, 일반 메소드, default 메소드, static 메소드 모두 포함

- protected 접근 지정 선언 불가

- 다중 상속 지원
