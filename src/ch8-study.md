1.예외처리
- 예외 : 프로그램 실행도중 예기치 않은 상황이 발생하는 것(오류와는 다름)
  
  문법적인 오류가 아닌 알고리즘에 문제이거나 변수(매개변수 포함)등의 값의 문제
  

- 예외처리 : 예외상황에서 예외메시지를 보여주는 것이 아닌 사용자가 알아볼 수 있는 메시지 등으로 처리

2. try ~ catch문 : 예외처리를 위해 존재하는 방법으로 하나의 try에 여러 개의 catch문을 지정할 수 있다.
- 예외가 발생하면 여러 개의 catch문중 발생한 예외를 처리하는 부분이 있으면 해당 catch문을 실행
- 여러 개의 catch문을 사용할 경우 상위클래스의 처리문을 하위클래스의 처리문보다 위에 지정하면 안 됨


      try {
               예외가 발생할 확률이 있는 실행문
               (일반적으로 실행문 전체를 입력하여 실행문이 중간 끊어져서 보이지 않게 처리)
      } catch  (예외종류1 예외 인스턴스명) {
               try문에서 실행한 실행문이 예외를 발생시켰을 때 처리할 구문
               지정한 예외가 아닌 다른 종류의 예외가 발생하면 처리 못 함
      }
      
      } catch (예외종류2 예외 인스턴스명) {
               try문에서 실행한 실행문이 예외를 발생시켰을 때 처리할 구문
               지정한 예외가 아닌 다른 종류의 예외가 발생하면 처리 못 함
      }
      
      } catch(Exception e){
               // 모든 예외클래스의 상위클래스
               모든 예외에 대한 처리구문
               // 마지막에 Exception클래스로 처리하게 되면 놓치는 예외는 없음
               // 다만, 어떤 종류의 예외인지는 알기 힘듦
      } finally {
               예외발생과 상관없이 반드시 실행되는 영역
               (return이나 continue키워드와도 상관없이 실행됨)
               // 사용시 반드시 필요한 구문인지 확인 후 사용해야 함
      }

3. 자주발생하는 예외
   
         ArrayIndexOutOfBoundsException
         - 배열의 인덱스값을 범위가 넘어서는 값으로 사용했을 경우
         
         ArithmeticException
         - 연산이 불가능한 상황(0으로 나누기)
         
         ClassCaseException
         - 허용할 수 없는 형변환이 발생했을 경우
         
         NagativeArraySizeException
         - 배열의 크기를 음수로 지정하면 발생
         
         NullPointerException
         - 생성되지 않은 상태의 인스턴스를 참조했을 경우

4.  함수에서 throws 키워드를 이용해서 예외를  넘길수도 있다.

아래과 같이 throws Exception으로 예외를 메소드 밖으로 넘겨버린 것이다. 객체를 사용할 때, try-catch로 예외 처리를 반드시 해줘야하는 객체일 때, throws 키워드를 이용해서 예외를 처리해줄 수도 있다.

      public static void main(String[] args) throws Exception {
      //코드 내용
      }
추가로 인위적으로 예외를 발생시킬 때 throw라는 키워드를 사용하는데 throw와 throws의 차이는

throw는 메소드 내에서 상위 블럭으로 예외를 넘기는 것이고,

throws는 현재 메소드에서 상위 메소드로 예외를 넘기는 것입니다.


인위적으로 익셉션을 발생시킬 때, 다음과 같이 Exception 클래스를 이용하여 익섹션 객체를 생성해서 넘겨줄 수 있다.

throw new Exception();
