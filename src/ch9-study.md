# 챕터9에 대해서 정리해보자

1.컬렉션 프레임웍

- 데이터의 묶음을 처리하는 여러 종류의 클래스들의 총칭
- 배열과 비슷한 개념으로 사용됨 (단, 기능은 많으나 속도가 느림)
- 배열과 가장 큰 차이점은 새로운 데이터를 추가할 수 있음
- List, Set, Map으로 구분되어 있음
- List : 순서가 있는 목록으로 중복값을 허용
- Set : 순서가 없는 집합으로 중복값을 허용하지 않음
- Map : 키(key)와 값(value)의 쌍으로 이루어진 집합(순서는 없고, 키는 중복불허, 값은 중복허용)
- jdk1.2이전의 Vector나 Hashtable등의 기능은 될 수 있으면 사용 안 함
  (Vector 대신 ArrayList, LinkedList등을 사용, Hashtable 대신 HashMap, TreeMap등을 사용)
